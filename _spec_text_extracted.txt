모드에서 강제값 또는 최소/최대 제한10.2 권장 수식FinalDamage = ((baseDamage + sumFlatDamage) x (1 + sumDamagePercentAdd)) x product(damageMultipliers)FinalAttackIntervalMs = max(minAttackIntervalMs, baseAttackIntervalMs / (1 + sumAttackSpeedPercentAdd) / product(attackSpeedMultipliers))FinalRange = clamp((baseRange + sumFlatRange) x product(rangeMultipliers), minRange, maxRange)FinalCritChance = clamp(baseCritChance + sumFlatCritChance + sumCritPercentAdd, 0, critCap)FinalSlow = clamp(baseSlow + sumFlatSlow + sumSlowPercentAdd, 0, slowCap)공격속도는 '초당 발사 수'보다 '공격 간격(interval)'을 내부 기준값으로 저장하는 편이 안전하다. 그래야 공격속도 증가가 중첩될 때 0 또는 음수 간격 문제가 발생하지 않고, 버프 해제 시에도 원래 값으로 안정적으로 돌아간다.10.3 예시 계산예: 펄스 터렛이 영구 레벨로 공격력 +24%, 건물 버프로 +5%, 전투 티어 업그레이드로 공격력 +15, 전술 모듈로 +10%, 지원 오라로 x1.08 배를 받는다고 가정한다. 최종 공격력 = ((42 + 15) x (1 + 0.24 + 0.05 + 0.10)) x 1.08 = 약 85.4 -> 표시값 85.10.4 영구 레벨업 / 인게임 업그레이드 분리 규칙영구 레벨업은 BattleStart 시 TowerSnapshot 으로 고정된 기본 수치를 만든다.인게임 업그레이드는 TowerRuntimeState 에만 존재하며 전투 종료 시 폐기한다.건물 패시브는 전투 시작 시 일부를 스냅샷으로 반영하고, 액티브/오라는 전투 중 동적으로 갱신한다.전투 중 재계산이 필요한 값은 range, damage, cooldownRemainingRatio, targetingFlags 정도로 제한한다.10.5 매각 규칙타워 매각 시 환불 금액은 '그 타워에 실제로 소비한 전투 골드 총합 x refundRate'로 계산한다. refundRate 기본값은 0.75다. 모드 또는 이벤트에 따라 0.70 - 0.80 사이 값을 적용할 수 있게 설정 파일로 분리한다.11. 기술 설계권장 기술 구조는 Flutter 를 앱 쉘과 로비 UI에 사용하고, Flame 을 전투 런타임에 사용하는 방식이다. 전투는 FlameGame 컴포넌트 트리 안에서 업데이트 / 렌더링하고, 로비와 설정, 상점, 랭킹, 관리 화면은
